CLIP - OpenStack Cloud Platform Infrastructure Overview

Information about the hardwe and ararchitecture CLIP

The CLIP platform a set of physical compute resources of differently configured compute nodes. These compute nodes are managed by a central control plane based on the “OpenStack” software framework for implementing private clouds.

CLIP Platform

OpenStack

OpenStack is an open source software framework that enables organisations to orchestrate all data center components (compute, network, storage, …) in a programatic way. OpenStack is used as a software framework to implement on premise local datacenter cloud computing services that are comparable in features to offerings from Amazon (AWS), Google (GCE), Microsoft (Azure), etc.

OpenStack can be imagined as an ‘Operating System for the Data Center’. Physical resources are being managed by OpenStack and applications are implemented on top of virtual machines running on the physical compute resources.

Advantages offered by OpenStack are

  • Physical resources can be pooled and applications can be carved out in more flexible and automated way
  • Applications are abstracted away from physical infrastructure
  • Offers an automation framework that talks to data center components in an orchestrated way
  • Enables longevity of applications as physical resoures can be swapped out transparently
  • A compute cluster can offer more services then just batch scheduling etc.

Physical Cluster ‘CLIP’

The physical resources available to the OpenStack framework are compute nodes of different resource configuration (CPUs, memory, SSDs, network interfaces, etc.). This allows for a flexible set of physical resources that can be tied together for various applications running in the cloud.

The node types are

  • (several) regular compute nodes – providing the bulk of the processor resources for hosting stateless and statefull applications and running OpenStack instances
  • flash dense IO nodes – used for amongst other things hosting a shared distributed parallel file system
  • memory dense nodes – provided for shared memory applications with high memory demand
  • GPU dense nodes – used for special purpose applications as accelerators

This infrastructure is deployed and managed by OpenStack running on dedicated service/controller nodes.

CLIP Architecture

Payload HPC Cluster ‘CBE’

User accessible services and applications are implemented on top of the cloud framework. OpenStack offers rich automation interfaces to provision applications in a programmatic way. The compute cluster CLIP Batch Environment or ‘CBE’ is one of those. Virtual machines in clouds are called instances.

Components of the system

  • Login nodes as instances
  • Compute nodes running the SLURM execution services as instances of various flavours matching physical resources
  • SLURM server instance running SLURM controller services
  • SLURM accounting DB server instances running SLURM database services
  • IO Nodes for parallel file system (BeegFS)
