Interactive workloads

This document describes how to submit interactive workloads

To ensure short start times for interactive workloads, we have created a recurring reservation for such jobs.

ReservationName=interactive StartTime=2020-03-18T09:00:00 EndTime=2020-03-18T17:00:00 Duration=08:00:00
   Nodes=clip-c1-[9-11],clip-c2-0 NodeCnt=4 CoreCnt=88 Features=(null) PartitionName=c Flags=FLEX,WEEKDAY,REPLACE,NO_HOLD_JOBS_AFTER_END
   TRES=cpu=88
   Users=(null) Accounts=gmi,imp,imba,vbcf,hephy,smi Licenses=(null) State=ACTIVE BurstBuffer=(null) Watts=n/a

The name of the reservation is “interactive“. Jobs from interactive services (RStudio, XPRA, Jupyterhub) will launch on this reservation, but also sbatch / srun submitted jobs can be sent to it.

Important notes

  • the reservation is ACTIVE from 0900 to 1700, repeating every WEEKDAY.
  • the reservation only contains nodes from “c” partition
  • jobs started within the reservation will continue running once the reservation becomes inactive
  • if enough idle resources are available jobs submitted to the reservation can also run on nodes not part of the reservation

How to submit interactive jobs

Keep your jobs as small as necessary (cpus, memory). Use –time to set appropriate time limits for your job. Use this resource responsibly!

# example interactive job
srun -n 1 --mem=1g --time=1:00:00 --reservation=interactive --pty bash

This will launch a job with:

  • 1 task (core), 1 GB memory, that will expire after 1 hour.
  • the resources will be taken from the reservation “interactive”
  • --pty attaches the terminal to the first process in the job (the bash shell), i.e. you prompt will continue inside the allocated job
