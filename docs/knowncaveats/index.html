Known Caveats

This page contains a list of known issues and how to solve or work around them

Batch Jobs

Graphical User Interfaces in Interactive Batch Sessions

When you enable X11 forwarding on your SSH connection to the cluster, you will be able to work with graphical user interfaces on the login nodes. When you work in an interactive batch session to keep load off the login nodes (which you should), the following approach is required:

ssh -Y username@cbe.vbc.ac.at
salloc bash -c 'ssh -Y $(scontrol show hostnames | head -n 1)'

### module load and run your application program, here it's R
module load r/3.5.1-foss-2018bÂ

RStudio

Package installation fails

Rstudio (via XPRA web interface) uses one of the normal R modules on CBE. In case the installation of custom R packages fails, try to install from the command line:

ssh username@cbe.vbc.ac.at
module load r/3.5.1-foss-2018b		### ... or whichever version Rstudio is using at the time

> install.packages("PackageName",dependencies = TRUE)

SSH connection issues to CBE

Windows 10 and WSL and VPN

If you use Windows 10 and a WSL (Windows Subsystem for Linux) terminal such as bash to connect via ssh to cbe while being connected with VPN you might see following error:

ssh cbe.vbc.ac.at -l username
ssh: Could not resolve hostname cbe.vbc.ac.at: Name or service not known

This is due to a bug in WSL which breaks the DNS name resolution of cbe.vbc.ac.at. There are currently 3 workarounds ranging from simple to advanced.

Solution 1 (simple):

Don’t use WSL but use the Windows 10 native terminal (cmd.exe) or the Putty ssh client to connect to cbe.

Solution 2 (medium):

Connect directly the IP of one of the CBE login nodes instead of the dns name.
First determine the IP of one of the CBE login nodes using the Windows 10 native terminal:

Find out IP address

Then connect using this IP:

$ ssh 172.24.96.13 -l username

In order to avoid remembering the IP address everytime you connect to CBE, you can add CBE to your ssh config. In the WSL terminal edit the file ~/.ssh/config (in case it doesn’t exist just create it) using vim or any other editor and add following section:

Replace username with your actual username 
Host cbe
    User username
    HostName 172.24.96.13

You can then connect to cbe.vbc.ac.at simply with:

$ ssh cbe

Solution 3 (advanced):

Add the campus DNS server to /etc/resolve.conf

Warning

You can potentially break DNS name resolution inside WSL. So be careful

Edit the /etc/resolve.conf file in your WSL terminal and add following line to the end of the section and save the file:

nameserver 172.16.80.12

Important

Beware that this nameserver is only reachable and thus working when connected via VPN

You can then connect to cbe using:

$ ssh cbe.vbc.ac.at -l username
